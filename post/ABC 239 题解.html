「AtCoder Beginner Contest 239」

# A - Horizon

  • 题目链接
  • 题意
    • 给出一个 HHH，求 H(12900000+H)\sqrt{H(12900000 + H)}H(12900000+H)​。
  • 题解
    • 直接用 sqrtsqrtsqrt 算即可。

# B - Integer Division

  • 题目链接

  • 题意

    • 给定一个整数 XXX 求 $\left \lfloor \frac {X}{10} \right \rfloor $。

  • 题解

    • 注意数据范围，需要 long long ，且需要判断是否为负数。如果是负数的话向下取整需要手动 −1-1−1，如果是正数，默认向下取整。

# C - Knight Fork

  • 题目链接
  • 题意
    • 给出两个点 (x1,y1),(x2,y2)(x1, y1), (x2, y2)(x1,y1),(x2,y2)，判断坐标系中是否恰好有一个点能同时距离这两个点 5\sqrt{5}5​，同时该点位于坐标系中的格子上。（即 xi,yix_i, y_ixi​,yi​ 均为整数）
  • 题解
    • 先判断两个点距离是否超过了 252\sqrt{5}25​ 。
    • 由于符合相交条件的两个点已经相距很近了，所以可以直接枚举判断是否有重合的点。

# D - Prime Sum Game

  • 题目链接
  • 题意
    • Takahashi 和 Aoki 正在玩一个游戏。Takahashi 从 AAA 到 BBB 中选择一个整数，并告诉 Aoki，Aoki 再从 CCC 到 DDD 中选择一个整数，如果这两个数的和是质数的话，则 Aoki 获胜，否则 Takahashi 获胜。
  • 题解
    • 首先预处理最大值范围内的质数。
    • 观察到数据范围很小，因此我们直接枚举每一种选数方案，假如 Takahashi 选择某一个数的时候，Aoki 无论选择哪一个数都不会使和为质数，则 Takahashi 获胜。

# E - Subtree K-th Max

  • 题目链接
  • 题意
    • 如题目字面意思，求出子树上第 kkk 大的数。
    • 给定一棵树，各节点按顺序排列编号，每个点都有自己的值，给出 N−1N-1N−1 条边连接各节点。
    • 有 QQQ 次询问，每次询问以 ViV_iVi​ 为根的子树上第 KKK 大的值。
  • 题解
    • 按题意连接各节点，无向边。
    • 预处理，从根节点开始直接跑一遍递归，将以各个节点为根节点的树的子节点的值排序后存起来，因此可以在 O(1)O(1)O(1) 的情况下查询值。
    • 时间复杂度 O(NKlog(NK))O(NKlog(NK))O(NKlog(NK))。
