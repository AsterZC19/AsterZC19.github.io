# CF451E | Devu and Flowers

  • 题意
    • Devu 有 nnn 个花瓶，第 iii 个花瓶里有 fif_ifi​ 朵花。他现在要选择 sss 朵花。
    • 你需要求出有多少种方案。两种方案不同当且仅当两种方案中至少有一个花瓶选择花的数量不同。
    • 答案对 109+710^9 + 7109+7 取模。
  • 分析
    • 假如没有花瓶中对花数量的限制，我们可以用隔板法分隔出来不同的方案，我们要在 sss 朵花中插入 n−1n - 1n−1 块板，第一块板有 (s+1)(s + 1)(s+1) 种选择，最后的总数是 (s+1)×(s+2)×⋯×(s+n−1)=(s+n−1)!s!(s + 1) \times (s + 2) \times \cdots \times (s + n - 1) = \frac{(s + n - 1)!}{s!}(s+1)×(s+2)×⋯×(s+n−1)=s!(s+n−1)!​ 。
    • 但是方案数量与板的顺序无关，所以我们还要去掉这些板全排列的数量，即 Cs+n−1n−1C^{n - 1}_{s + n - 1}Cs+n−1n−1​ 。
    • 现在加入条件，我们只要把不符合条件的方案减去即可。
    • 首先我们要从所有的花中取出 fi+1f_i + 1fi​+1 朵在花瓶 iii 中的花，这样无论如何都是不符合条件的，方案数是 Cs+n−1−(fi+1)n−1C^{n - 1}_{s + n - 1 - (f_i + 1)}Cs+n−1−(fi​+1)n−1​。
    • 之后就考虑使用容斥原理，因为我们之前减掉的方案有重复的，又因为有 1≤n≤201 \le n \le 201≤n≤20，我们可以用状态压缩枚举每种情况，又因为组合数范围很大，我们直接使用逆元求组合数。

# SP32952 | ADAFTBLL - Ada and Football

  • 题意
    • 你有一棵共 nnn 个节点的树，一共有 qqq 次操作，操作有两种分别为：
      1. 1 x y ：表示把第 xxx 个节点的权值变成 yyy。
      2. 2 u v ：输出 ∑s×(s−1)2\sum \frac{s \times (s - 1)}{2}∑2s×(s−1)​，其中 sss 为 uuu 到 vvv 节点中出现的不同权值的次数。
  • 分析
    • 树上带修莫队。
    • 树上莫队的宗旨是把树变成序列，就能按普通序列处理了，这里要用到两个知识点 ：重链剖分 + 欧拉序。
    • 对于 a−ba - ba−b 的路径，分两种情况讨论。
      • aaa 和 bbb 在一条链上，途中的节点都只在 aaa 和 bbb 第一次出现的 dfs 序之间。
      • aaa 和 bbb 不在一条链上，那途中节点则为 aaa 最后一次出现的地方到 bbb 第一次出现的地方之间只出现一次的节点，aaa 最后一次出现因为把 aaa 的子树遍历完了， 之所以只出现一次和上面一样，不过最后还要加上它们的 lcalcalca，因为 a 和 b 都在 lca(a,b)lca(a, b)lca(a,b) 的子树里，lcalcalca 没有遍历到。
    • 注意，需要开一个数组记录节点的访问次数，如果访问一次就 del，没有就 add，然后取反即可。

# AT5140 [AGC035C] Skolem XOR Tree

  • 题意
    • 给定一个正整数 NNN，试判断，是否存在这样一棵节点数为 2N2N2N 的树，满足：
      • ∀i∈[1,N]\forall i \in [1, N]∀i∈[1,N]，第 iii 号节点和第 i+Ni + Ni+N 号节点的权值均为 iii。
      • 第 iii 号节点到第 i+Ni+ Ni+N 号节点路径上的点的点权异或和恰为 iii。
  • 分析
    • 下文 x′x'x′ 代表 xxx 的另外一个点，即 x+Nx + Nx+N，值仍为 xxx。
    • 首先考虑无解的情况，当 lowbit(n)=nlowbit(n) = nlowbit(n)=n 的时候无解，因为这样的话会存在 nnn 与 n′n'n′，除了这两个数以外找不到其他的数的异或和为 nnn ，因此无法匹配。
    • 之后考虑其他的情况，分奇数偶数考虑，有一个好用的性质：

    • x⊕1=x+1(xisanevennumber)x \oplus 1 = x + 1 (x \; is \; an \; even \; number)x⊕1=x+1(xisanevennumber)

    • 因此，如果我们假设 111 节点为根，就可以发现，将所有的数对 (x,x+1)(x, x + 1)(x,x+1) （其中 xxx 为偶数并且 x+1≤nx + 1 \le nx+1≤n ）匹配到一起就比较好处理。我们可以找到路径 x→x+1→1→x′x \to x + 1 \to 1 \to x'x→x+1→1→x′ 和 x+1→1→x′→(x+1)′x + 1 \to 1 \to x' \to (x + 1)'x+1→1→x′→(x+1)′ 。
    • 接下来还有两个问题需要处理。
      • 作为根的 111 没有匹配，如何加入 1′1'1′ ？
      • 假设 111 为根，随便找一个叶子接上 1′1'1′，可以找到路径 1→x→(x⊕1)→1′1 \to x \to (x \oplus 1) \to 1'1→x→(x⊕1)→1′。
      • 如果 nnn 为偶数，我们按照上面的匹配完就会发现 nnn 以及 n′n'n′ 还没连上，如何处理？
      • 我们依然在根上处理，因为无解情况排除了 nnn 在转化成二进制后只存在一个 111，于是我们可以找到一条路径 n→lowbit(n)→1→(n−lowbit(n)+1)′→n′n \to lowbit(n) \to 1 \to (n - lowbit(n) + 1)' \to n'n→lowbit(n)→1→(n−lowbit(n)+1)′→n′。

    • 不妨假设 x<y<zx < y < zx<y<z。

# 2022 ICPC Online Contest (Ⅰ) | L. LCS-like Problem

  • 题意
    • 给定两个字符串 s,t(∣s∣,∣t∣≤5×105)s,t \; (|s|,|t| \le 5 \times 10^5)s,t(∣s∣,∣t∣≤5×105) 。
    • 你需要在 sss 中找到一个最长的子序列 s′s's′，满足 s′s's′ 和 ttt 的任意一个公共子序列，长度都不超过 111 。
    • 输出 s′s's′ 的长度。
  • 分析
    • 先预处理不能接上的可能， ban[a][b] = 1 ，表示在选择的 s′s's′ 中，不能出现 a⋯ba \cdots ba⋯b 这种情况。
    • 我们定义 dp[i][j]dp[i][j]dp[i][j] 为 [1,i][1,i][1,i] 中最后字母为 jjj 的（jjj 在 ttt 中出现过）可以组成 s′s's′ 的最大长度。
    • 枚举 s[i]s[i]s[i] 放到某个 jjj 后面，我们需要保证可以转移。
    • 但是我们这样看上去只是保证了相邻的两个字符没有被 ban，假如有不相邻的字符被 ban 了呢？
    • 我们假设 ban[a][b] = 0, ban[b][c] = 0, ban[a][c] = 1
    • 我们考虑下是否会有这种情况。因为 ban[a][c] = 1 ，所以在 ttt 中，aaa 一定在 ccc 前面。所以我们考虑 bbb 的位置。
    • 这所有三种情况都不会成功转移，所以只需要考虑两个相邻的会不会被 ban。

# SP1805 HISTOGRA - Largest Rectangle in a Histogram

  • 题意
    • 在一条水平线上有 nnn 个宽为 111 的矩形，求包含于这些矩形的最大子矩形面积。
  • 分析
    • 经过观察，限制每一个大矩形的高度的都是组成这个大矩形中最矮的那个矩形决定的。
    • 所以，假如往后的矩形高度都是非递减的，那么我们可以一直往后加，如果下一个高度更小了，那么加入它所构成的矩形高度不可能超过它自己，我们可以这样模拟一遍，更新比当前新的小矩形的高度高的答案再将他们合并。
    • 我们使用单调栈来维护。

# SP29455 AROPE - Alphabetic Rope

  • 题意
    • 维护一个字符串，支持以下 333 种操作：
      • 1 X Y ：把区间 [X,Y][X, Y][X,Y] 剪切下来连接到字符串开头。
      • 2 X Y ：把区间 [X,Y][X, Y][X,Y] 剪切下来连接到字符串末尾。
      • 3 Y ：查询当前字符串中第 YYY 个字符，输出到单独一行。
  • 分析
    • 可以用 Splay 区间翻转完成该题，111 操作就只需要先翻转 [L,R][L, R][L,R]，接着翻转 [1,L−1][1, L - 1][1,L−1]，然后再翻转 [1,R][1, R][1,R]。222 操作同理。
